## Why StateBase?

We replace brittle "Prompt Engineering" with deterministic **State Engineering**.

| Feature | StateBase | DIY / Vector DB |
| :--- | :--- | :--- |
| **Persistence** | **Sessions** (Immutable Ledger) | Ephemeral Logs |
| **Context** | **Graph + Vector** (Semantic) | Naive Chunks |
| **Logic** | **State Machines** | Probabilistic |
| **Scale** | **Serverless** | Self-Managed |
